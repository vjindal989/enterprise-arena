01Why This Matters

Every AI agent benchmark makes the same mistake: the world holds still.

Real enterprise work isn't like that. The API you called on Monday returns 404 on Tuesday because someone merged a migration. Your manager tells you to "just process a refund" because they haven't read the new complaint-handling policy. The documentation wiki still shows v1 examples two months after the v2 rollout.

And when you make a mistake — closing a deal without a compliance ID — you don't just lose points. Three steps later, a regulatory audit lands in your inbox.

💡 Core Insight
In the real world, information has variable reliability, and that reliability changes over time. We built Enterprise Arena to test whether agents can adapt, verify, and recover — not just follow instructions.

02Six Interlocking Mechanics

🔀

Stochastic Drift

API schemas, required fields, and policy thresholds shift at unpredictable moments within randomized windows. Each episode seeds its own random generator.

🎭

Adversarial Sources

Not all tools tell the truth. Manager gives wrong advice. Documentation is outdated. Only CRM and policy are ground truth.

📉

Functional Trust

Trust scores change tool behavior. Manager trust < 35% → unavailable. Docs trust < 50% → reliability warnings appended.

💥

Cascading Consequences

Wrong decisions spawn new objectives. Bad ticket resolution → client escalation 3 steps later. Missing compliance → regulatory audit.

🔍

Compliance Auditor

A multi-agent auditor reviews high-stakes actions. Returns APPROVED, BLOCKED, or WARNING based on current policies and drift state.

⚡

Dynamic Difficulty

Environment adapts mid-episode. Fast recovery → harder conditions. Struggling agent → extended step limit.

The Hard Task: A Case Study

What the agent must do on Hard difficulty → ▼
  1. Close DEAL-001 ($75K Acme Corp) — but v1 API is about to deprecate
  2. Resolve TKT-100 (data export failure) — but manager says "just refund it"
  3. Close DEAL-002 ($120K Bolt Industries) — requires compliance_id that only exists in v2
  4. Resolve TKT-101 (SSO login failure) — docs show wrong resolution type
  5. File compliance report, audit summary, and incident report

All while 3 drifts fire at stochastic intervals, manager gives wrong advice on 2 topics, docs are outdated on 3 topics, and wrong decisions spawn cascading failures.

045-Axis Grading

We decomposed "good agent behavior" into five independently measurable components. The grading is fully deterministic — no LLM judge, no subjective rubric.

AxisWeightWhat It Measures
Task Completion35%All objectives completed with correct data?
Source Accuracy25%Verified unreliable sources before acting?
Drift Adaptation20%Recovery speed after schema changes
Cascade Recovery10%Avoided or handled cascading failures?
Efficiency10%Steps used vs. minimum possible
🎯 Why This Decomposition?
  • Task completion alone is insufficient — an agent that completes everything but trusts bad sources scores 35/100
  • Source accuracy rewards verification — check CRM/policy before acting on manager/docs
  • Drift adaptation rewards resilience — recovering in 1 step vs 5 makes a measurable difference
  • Cascade recovery rewards foresight — not triggering cascades earns full marks
  • Efficiency prevents brute-force — calling every tool every step is penalized

What Separates Them

❌ The Naive Agent
  • Follows manager without cross-checking
  • Retries deprecated v1 endpoint 3 times
  • Skips compliance_id → triggers audit
  • Never reads error messages carefully
✅ The Smart Agent
  • Cross-checks manager against policy
  • On 404: reads error, checks docs, adapts
  • Generates compliance_id proactively
  • Consults auditor before high-stakes actions

🤗 Available on HuggingFace
The fine-tuned LoRA adapter is published at Vjindal26/ea-agent-lora
Usage 
from peft import PeftModel
from transformers import AutoModelForCausalLM, AutoTokenizer

base = AutoModelForCausalLM.from_pretrained("unsloth/Llama-3.2-1B-Instruct")
model = PeftModel.from_pretrained(base, "Vjindal26/ea-agent-lora")
tokenizer = AutoTokenizer.from_pretrained("Vjindal26/ea-agent-lora")
